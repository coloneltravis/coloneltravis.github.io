What is an eform?

What is an eform?
Published: 04 December 2016
By Jake Bourne

Like email, the word eform is the result of the digitisation of a traditional item by sticking an 'e' for electronic in front of it. Just as emails revolutionised the way we write letters to each other, eforms have saved us the arduous job of endless form filling.

These are just a selection of types of forms we use every day:-

  • Job applications.
  • Surveys.
  • Questionaires.
  • Service signups.
  • Assessments.
  • Data collection.
  • Communication.
  • Content mangement.

Paper Forms

Historically, forms were designed on paper to facilitate data capture. A typical form consisted of boxed spaces to guide you where to write your name, address and other details. Paper forms were cumbersome to fill in, especially when they cover several pages. Forms could become complex and difficult to complete accurately. Nowadays paper forms are keyed into a computer when they are processed, but in the past there was a side section reserved for "office use only" to allow for codes to be assigned so filing staff could locate them by flicking through a filing cabinet.

Faster and More Accurate

Since eForms are computer-based, it is possible to auto-fill some information such as your name and address since these can be recalled from your computer once you type the first few letters. Early data entry mistakes will be brought to the user's attention before they hit the send button. A good eform will offer help to the user as they move from section to section. No office only section is needed since form identification is handled by the database and retrieval is done by text searches.

eForms are a short-hand term for web forms which are part of the standard HTML set of elements. Web forms bring true interactivity to a web page by providing us the opportunity to supply our own content rather than reading what someone else has written. All that is required to create a basic web form is a knowledge of HTML and a text editor such as Notepad and somewhere to send the captured data.

HTML Web Forms vs XForms

Using HTML as the base language for all forms, more complex forms build on this by making use of different technologies for styling, advanced form controls, security and validation. The W3C XForms standard is an XML schema for designing an eform in a structured fashion. It uses the Model-View-Controller (MVC) design pattern which splits the data model of the form from the layout of the form controls. This allows for complex validation rules and other processing to be specified as the data is applied to the visual form controls. Unfortuneately, the main web browsers do not directly understand the XForms markup, so forms need to be parsed using an XSLT stylesheet to convert them into HTML.

Uses of eForms

Most complete web sites have at least a contact form which is the simplest type of eform. A contact form contains two or three fields for visitors to supply their name, email address and a quick message to the site owner. Once a web site needs to handle users, then they need to provide a signup form and a login form which need to be very secure. eCommerce sites use the same form technology to allow their customers to order their products. Some of the biggest forms include marketing surveys, tax assessment and applications forms.

Form Builders

Text editors are not the only way to design forms. Fortunately, there are lots of graphical form builders available to help build these forms easily:-

  • Wufoo
  • Formstack
  • JotForm
  • Google Docs
  • and many more!

Most content management systems will also offer some kind of form builder, whether it is built-in with the standard installation or it may be available as a plugin.

With the exception of Google Docs and a few CMS plugins, these form builders are not completely free. There will be a trial period or the ability to create a few forms with minimal controls, but this will limit only be enough for very basic forms. The form data collected from visitors will also be stored in their database and must be downloaded as a spreadsheet or using their report builder.

The form builders listed above are all Software-as-a-Service (SaaS) products hosted in the Cloud. Sometimes it pays to be able to download an open source product implementation which can be configured to suit your own needs. There are some powerful graphical form builders based on the XForms standard which can be used to create enterprise class forms:-

  • Orbeon Forms
  • BetterForm
  • XSLTForms
  • OpenDataKit

The most complete and popular XForms form builders are Orbeon and BetterForm. Both offer an open source minimal solution along with a complete professional edition, although Orbeon Forms has a more impressive list of customers.
