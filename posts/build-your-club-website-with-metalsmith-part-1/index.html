Build Your Club Website with Metalsmith - Part 1

Build Your Club Website with Metalsmith - Part 1
Published: 15 November 2016
By Jake Bourne

Most web officers of new clubs tend to opt for a CMS like Wordpress to manage their club's website. Some may choose a hosted solution from the likes of Weebly or Wix. However, what they fail to realise is that a CMS resides in a database which can be sluggish with a high volume of users and open to attacks by hackers.

Metalsmith is a static website generator which allows fast and secure static web sites to be generated as simply as with a Content Management System (CMS).

All web pages are pre-built from source files so they can be copied to a web server. Metalsmith is, in effect, a file processor since it takes in a set of input files, transforms them and produces output files.

As no database is required it removes a major security threat and takes an extra hop out of the process to access a web page.

Traditionally, building a static web site involved typing and copying lots of HTML and CSS code. Metalsmith reduces this task to the creation of layout templates which can then be merged with the content itself to produce the final page.

The majority web sites do not require a complete CMS manage their content. A database is designed to handle normalised data records for people management and stock control systems. A CMS database stores large amounts of unstructured text and binary images.

Before you write off static web sites as too difficult to implement, have a read of this how-to guide on how to build your website with Metalsmith.

Installing Metalsmith

Metalsmith was developed in Node.js which is a modern language based on Javascript. If you don't already have it installed on your system, head to https://nodejs.org/ to download it, extract the archive and run the install scripts.

Once Node.js is installed, open a Command Prompt window. Node.js is a CLI (command-line interface) language like Ruby and Python. Node.js uses the Node Package Manager (npm) to install and manage packages.

Let's get started and create the structure for our club website.

  • In a suitable folder, create a new folder: mkdir myclubwebsite

  • Now go into the new folder and run the command: npm init

    This will initialise a Node.js application by creating a package.json file, which contains main configuration information for your app.

  • Create a sub-folder to store all your source files: mkdir src

  • Create a sub-folder where Metalsmith can place the final build of your website: mkdir build
  • You are now ready to install the Metalsmith package: npm install metalsmith --save-dev

The --save-dev option will save the package in a sub-folder and keep a record of in the package.json file.

Setting up your website

One of the unique features of Metalsmith is that is that everything is a plugin connected to each other. Each plugin is chained to its preceding one which passes it a full list of files and application data to perform its task. Such tasks may include converting Markdown to HTML, resizing images or processing CSS. But first, we need to create a build script to build our website.

Open your favourite text editor and type the following code:- 

var Metalsmith = require('metalsmith'),
var markdown = require('metalsmith-markdown'),
var pug = require('metalsmith-pug'),
var layouts = require('metalsmith-layouts'),
var serve = require('metalsmith-serve');

Metalsmith(__dirname)
    .source('./src')
    .use(markdown())
    .use(layouts('pug'))
    .use(serve({port:8080}))
    .destination('./build')
    .build(function (err) { if(err) console.log(err) })

Save the code in a file called build.js in the myclubwebsite folder.

You will notice that there are a few extra plugins which we have not yet installed. Let's do that now by using npm install

npm install metalsmith-markdown --save-dev
npm install metalsmith-pug --save-dev
npm install metalsmith-layouts --save-dev
npm install metalsmith-serve --save-dev

Now, you may be wondering what actions each of these plugins do.

Markdown is a simplified markup language which you will use to edit your site content. The metalsmith-markdown plugin will process your Markdown files and turn them into HTML so they can be viewed in a web browser.

Pug is a powerful templating language which you will use to define the layout of your site. Pug will analyse your source files and arrange them according to their content type. The metalsmith-layouts and metalsmith-pug plugins work together. There are also other templating languages which could be used instead, such as Handlebars or Liquid.

The last plugin, metalsmith-serve, will provide you with a local web server which you can use immediately to view your website by navigating to http://localhost:8080/ in a web browser.

You are now ready to build your website by running the build.js script with the Node interpreter. Type and run node build.js on the command line.

There's not much to see yet since we have not created any content. In the next section we will start to plan how to layout our club website.
